# Review PRD

Relis et audite un fichier PRD existant pour vérifier la qualité, la complétude et la cohérence des user stories, acceptance criteria, et références MCP/skills.

## FR

```
Relis le PRD @tasks/prd-backend-hardening.md et fais un audit complet : vérifie la cohérence des user stories, la complétude des acceptance criteria, les dépendances entre US, les quality gates, et les références MCP/skills. Utilise context7 pour valider que les recommandations techniques sont à jour. Propose ensuite les corrections et améliorations directement dans le fichier.
```
